## 3. 测试执行方法

### 3.1 在DevEco Studio中执行测试

#### 方法1: 通过IDE运行
1. 打开DevEco Studio
2. 打开项目 `MyApplication05`
3. 在项目视图中找到测试文件：
   - `entry/src/ohosTest/ets/test/SettingsPage.test.ets`
4. 右键点击测试文件
5. 选择 "Run 'SettingsPageTest'"
6. 等待测试执行完成
7. 查看测试结果

#### 方法2: 通过测试类运行
1. 打开测试文件 `SettingsPage.test.ets`
2. 找到测试类 `SettingsPageTest`
3. 点击类名旁边的绿色运行按钮
4. 选择 "Run 'SettingsPageTest'"
5. 等待测试执行完成

#### 方法3: 运行单个测试用例
1. 打开测试文件 `SettingsPage.test.ets`
2. 找到要运行的测试用例（`it()` 函数）
3. 点击测试用例旁边的绿色运行按钮
4. 选择 "Run" 选项
5. 等待测试执行完成

### 3.2 通过命令行执行测试

#### 运行所有测试
```bash
# 在项目根目录执行
hvigorw test
```

#### 运行特定模块的测试
```bash
# 运行entry模块的测试
hvigorw test --module entry

# 运行ohosTest类型的测试
hvigorw test --module entry --test-type ohosTest
```

#### 运行特定测试类
```bash
# 运行SettingsPageTest测试类
hvigorw test --class SettingsPageTest
```

#### 生成测试报告
```bash
# 运行测试并生成报告
hvigorw test --report

# 查看测试报告
hvigorw test --report --open
```

### 3.3 在真机上执行测试

#### 准备工作
1. 连接鸿蒙真机到开发机
2. 在真机上启用开发者模式
3. 在DevEco Studio中选择真机作为运行设备

#### 执行测试
1. 按照上述方法执行测试
2. 测试将自动安装到真机并执行
3. 测试结果将显示在DevEco Studio的测试窗口中

## 7. 常见问题

### 7.1 测试失败怎么办？
1. 查看测试失败日志
2. 检查错误信息和堆栈跟踪
3. 确认测试环境是否正确
4. 检查数据库连接是否正常
5. 查看相关代码是否有问题

### 7.2 测试超时怎么办？
1. 检查设备性能
2. 检查网络连接
3. 检查数据库操作是否正常
4. 考虑增加超时时间

### 7.3 测试无法运行怎么办？
1. 确认DevEco Studio版本是否正确
2. 确认HarmonyOS SDK是否正确安装
3. 确认测试设备是否连接
4. 清理项目缓存：`Build > Clean Project`
5. 重新构建项目：`Build > Rebuild Project`

### 7.4 如何调试测试？
1. 在测试代码中设置断点
2. 右键点击测试用例
3. 选择 "Debug '测试用例名'"
4. 使用调试器逐步执行

## 8. 测试最佳实践

### 8.1 测试编写原则
- **独立性**: 每个测试用例应该独立运行
- **可重复性**: 测试结果应该可重复
- **清晰性**: 测试名称和描述应该清晰
- **完整性**: 测试应该覆盖所有功能点
- **快速性**: 测试应该快速执行

### 8.2 测试维护
- **定期更新**: 随着代码变更更新测试
- **删除过时测试**: 删除不再需要的测试
- **重构测试**: 保持测试代码整洁
- **文档更新**: 更新测试文档

### 8.3 持续集成
- **自动化测试**: 在CI/CD流程中运行测试
- **测试报告**: 生成测试报告并通知团队
- **失败处理**: 测试失败时阻止合并
